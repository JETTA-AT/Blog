Spark函数:浅析combineByKey、reduceByKey、groupByKey

Posted on 2019-02-21 | In Spark |

简单来说reduceByKey、groupByKey都是由combineByKey核心泛型函数实现的，reduceByKey是聚合函数，groupByKey是分组函数，让我们用代码和平均数求解问题，来认识这三个重要的Spark函数。

combineByKey函数

def combineByKey[C](  
      createCombiner: V => C,  
      mergeValue: (C, V) => C,  
      mergeCombiners: (C, C) => C,  
      partitioner: Partitioner,  
      mapSideCombine: Boolean = true,  
      serializer: Serializer = null)

三个自定义方法

1.createCombiner

这个函数把当前rdd中的值（value）作为参数，此时我们可以对其做些附加操作(类型转换)并把它返回 (这一步类似于初始化操作，分区内操作)

2. mergeValue

该函数把元素V合并到之前的元素C(createCombiner)上 (每个分区内合并)

3. mergeCombiner

该函数把2个元素C合并 (此函数作用范围在rdd的不同分区间内，跨分区合并)

使用combineByKey求平均数

测试代码如下：

import org.apache.spark.{SparkConf, SparkContext}

object combineByKey {
  def main(args: Array[String]): Unit = {
    val sparkConf = new SparkConf()
      .set("spark.io.compression.codec", "snappy")
      .setAppName("dsp_get_request").setMaster("local[*]")
    val sc = new SparkContext(sparkConf)

    val initialScores = Array(("Fred", 88.0), ("Fred", 95.0), ("Fred", 91.0), ("Wilma", 93.0), ("Wilma", 95.0), ("Wilma", 98.0))
    val d1 = sc.parallelize(initialScores)
    type MVType = (Int, Double) //定义一个元组类型(科目计数器,分数)
    val rdd2 = d1.combineByKey(
      score => (1,score),  // 将score映射为一个元组，作为分区内聚合初始值
      (c1: MVType , newscore) => (c1._1 + 1 , c1._2 + newscore),  //分区内聚合
      (c1: MVType ,c2: MVType) => (c1._1 + c2._1,c1._2 + c2._2)  //分区间聚合
    ).map{ case(name ,(num ,cnt)) => (name ,num / cnt)}
    rdd2.foreach(println)
  }
}

(Wilma,95.33333333333333)

使用reduceByKey求平均数

现在用reduceByKey改写刚才的平均数算法

    val rdd3 = d1.map(a => (a._1, (a._2, 1)))
      .reduceByKey((a,b) => (a._1+b._1,a._2+b._2))
      .map(t => (t._1,t._2._1/t._2._2))
    rdd3.foreach(println)

reduceByKey、groupByKey

我们再看看reduceByKey、groupByKey的区别：

d1.foreach(println)
------------------------
(Wilma,93.0)
(Fred,95.0)
(Fred,88.0)
(Fred,91.0)
(Wilma,95.0)
(Wilma,98.0)

d1.reduceByKey(_ + _).foreach(println)
-------------------------
(Wilma,286.0)
(Fred,274.0)

d1.groupByKey().foreach(println)
-------------------------
(Wilma,CompactBuffer(93.0, 95.0, 98.0))
(Fred,CompactBuffer(88.0, 95.0, 91.0))

reduceByKey效率会更高

其中reduceByKey(_ + _)写法同等于reduceByKey((a,b) => (a + b))
groupByKey会把所有的键值对集合都加载到内存中存储计算，若一个键值对太多，则会导致内存溢出。
reduceByKey 进行shuffle之前会在map做合并，这样就会减少shuffle 的IO 传送，效率会高一些。

groupByKey聚合

d1.groupByKey().map(t => (t._1,t._2.sum)).foreach(println)
-------------------------
(Fred,274.0)
(Wilma,286.0)
